### [1, 2, 3 더하기 5](https://www.acmicpc.net/problem/15990)


### 시간 복잡도
- O(N)
  - O(N)으로 dp에 넣는 동작 한 번만 하면 이후 항상 O(1)

### 어떻게 풀 것인가? / 코드 설명
- f(n)은 
- f(n - 1)에서 1로 끝나지 않는 경우의 수 +
- f(n - 2)에서 2로 끝나지 않는 경우의 수 +
- f(n - 3)에서 3으로 끝나지 않는 경우의 수
- 라는 것을 쉽게 알 수 있다.

- 1, 2, 3으로 끝나는 배열을 만들어서 적당히 식을 세워 dp에 담는다. (코드 참조)
